【小俊】我们接下来做一轮long context的长文本对话。假设你是一个大模型，你现在能支持的上下文窗口有多大？你看Google和OpenAI现在都能支持的是100万个token。

【李想】我觉得人类整个的生理特点，和人工智能是有本质差别的。人类并不擅长处理特别复杂的信息，所以我们经常讲人类要做"商紧"，而不要去做"商增"。我觉得人工智能非常擅长处理多么庞大的信息，包括今天预训练的规模，这个数据规模也不是15T了，因为现在已经开始有30T了，最新的Llama就是这样的。所以这是一个本质的差异。从这个角度而言，包括人类发明方法论、人类使用各种工具，都是为了减少能量的消耗，因为大脑也要消耗能量的。这是人类的一个特点。所以从本质上，人类和AI应该是个非常好的友好关系，各自做各自擅长的事情。

【李想】：Agent最重要的评判条件，就是它是否是个生产工具，它是否真正能替代我去完成专业的工作，它是否真的在产生有效的生产力，或者它是否真的在解决我工作中最重要的8小时的时间。我觉得这其实是我认为接下来对Agent最重要的一个衡量，也是Agent的意义所在。

【李想】：对。我觉得人工智能变成生产工具，然后才是真正人工智能爆发的时刻。

【李想】：我觉得生产工具的要素是什么呢？怎么样能够成为一个生产工具？它必须有action，不能只是"知"，必须"行"，知行合一才可以。就是今天你用o2、o3，多么聪明；你用DeepSeek R1，多么聪明——但其实它并没有真正地去行动，它只是给出策略，并做策略的推演。我工作的时候，我还是要通过操作电脑去处理很多工作，或者通过物理世界的东西去处理工作。所以我觉得它必须得有行动，行动才能产生价值。

【小俊】：对，所以像Manus，其实它就已经在行动上……

【小俊】：做了很多工作了。其实包括自动驾驶，本质上是控制一个机器在物理世界去行动。我觉得必须得能够行动，然后才可以……你怎么看Manus这个产品？

【李想】：我自己觉得，它是往生产工具的方向走。从整个产品的创意上来看，是往生产工具走得最大的一步。包括他们通过训练机，包括去调用工具。比如今天有一个挺大的问题——就是至少很长一段时间，所有的模型公司都认为其实不需要软件、不需要工具，做好模型就能够替代掉一切。

我拿人举个例子：你比我聪明十倍，但你拿勺子挖一个坑，我拿铲子挖一个坑——我再笨，效率都比你高。要是我比你聪明十倍，我更不会拿手去挖。工具意味着更高的确定性，以及更低的能量和脑力消耗。很多时候这两个并不冲突——更好的大脑和使用更好的工具并不冲突，而且脑子变强以后，工具也可以变得更强。

其实Manus从调用工具的角度去完成各种工作——比如分析特斯拉股票，去浏览SEC的网站，去浏览特斯拉投资者关系的网站，直接去阅读最顶级的各个投行分析师关注特斯拉的分析报告，而不是只是简单通过一个RAG然后作为信息的来源。我觉得这个更像是真正的专业工作人员和生产工具应该做事的方式。只是他们作为一个通用产品，覆盖的面太广了，这就是个很大的挑战。

【小俊】：为什么这样的一个Agent，可能是一个创业公司做出来的？你有想过吗？

【李想】：我个人感觉，在一个新技术产生的时候，反而会造成过去的大公司——或者说创业公司里已经做大的公司——把分工分得太精细了。但往往一个好产品，是多个东西组合在一起才能成立的。如果做模型的人认为"我就要做好模型，用模型解决一切问题"，这就会变成非常大的一个挑战。

【小俊】：我刚才跟你聊的是过去130天理想汽车的进步是什么，你回答了我很多关于你最新的思考。那你觉得，过去130天理想这家公司最大的进步是什么？它有没有成为一个更智能的组织？

【李想】：我觉得我们没有那么快的变化。就刚刚才讲的，其实这是一些重要的突破。然后我们要想，我们作为一个其实3万多人的企业，如果要发生很多重要的变化，还是得围绕真正的用户价值能够发生的变化去产生变化，不能只是说技术发生了一些变化，我就去改变我的组织。

我自己来看所有的变化之间的关系——第一，我会把规模放到一个中心。因为规模是个确定性的，我做1000亿收入还是做2000亿的收入。第二，是在围绕这个规模外边，其实有三个动态的变量：一个是用户需求会发生的变化，一个是技术产品发生的变化，还有一个是组织发生的变化。然后这三个之间的变化，我会相互诊断。

如果只是技术产品发生变化，但用户需求没发生变化，其实我并不会上来就去调整组织。很多时候是当我看到用户需求在发生变化的时候，产品技术的变化和用户需求变化产生了结合的时候，组织变化就必须跟上。所以这是我们来看的——我们不能因为今天出了什么东西，就明天调整一下，这肯定是有问题的。

所以过去130天，其实最大的变化是外部的环境。国际环境在发生重大的变化，本身的技术也在发生变化。但技术最大的变化还是中国带来的变化，就是中国在人工智能方面带来的变化。然后美国的变化，反而没那么大。

【小俊】：你从DeepSeek身上学到的是什么？

【李想】：我觉得DeepSeek我能学到的最好的一点，是它运用了人类的最佳实践——它极简地运用了人类最佳实践。比如说，它在做DeepSeek V3的时候，V3是一个MoE的6710亿参数的模型。我觉得MoE是个非常好的架构，它相当于把一堆专家组合在一起，然后每一个是一个专家能力。包括它在构建这个专家能力的方式，其实挺清晰的。

当我们想去构建能力的时候，DeepSeek已经展示了一个最佳实践——DeepSeek一定要先搞研究，我觉得这是非常重要的。就是任何时候，当我们想要去改变能力和提升能力的时候，DeepSeek是先搞研究，搞完研究以后才搞研发，搞完研发以后第三步是要把能力表达出来，第四步是把能力变成业务的价值。所以这四个步骤，是个极简的人类最佳实践。

但我们经常会忘掉这个——看到什么东西就直接去启动研发，而没有去搞研究。我们自己也很受益，其实我们是在端到端和VLM上，包括今天做VLA的时候，我们的研究团队其实表现得非常好。包括你可以看，国外像李飞飞她在引用自动驾驶研究的时候——

【李想】：也经常会引用我们关于自动驾驶方面的研究论文。我觉得这个其实很重要——研究跑通了以后，研发的效率会变得非常高，但是研发又非常在意价值，它能够把价值表达出来，然后变成业务。所以这是一个很重要的过程。

比如说我再举一个例子。大家如果用过我们的端到端，我们是端到端里唯一给你展示端到端是怎么工作的——端到端是怎么来取轨迹的，端到端的 attention 注意力是怎么工作的，VLM 是怎么工作的。其实这本身是你能力的一个展示，能让你的使用者或者所有的参与者都能够理解。我觉得这是一方面——这个交互和产品，是今天有了模型之后技术才能实现的，并不是简单靠产品能实现的。

【小俊】：对。

【李想】：我觉得这是做得好的。我觉得 DeepSeek V3 是一个表现，DeepSeek R1 又追寻了另外一个方向——它是个 reasoning，它是个推理模型。说白了，其实它就是业务。对吧？它其实就是业务，而业务没有用户就不是业务。

但是你可以看，从它整个思维链里边来看，它又追寻了一个人类做业务的极简最佳实践。第一步是什么呢？第一步其实是先做需求分析——用户给你一个需求，先做需求分析。第二步是什么呢？第二步其实是来确定目标，因为很多时候给的是一个提示，给的是一个方向，不一定会给出直接目标，所以它通过分析完以后，再来制定接下来要推理的目标。第三步，它会给出策略——当然作为一个 reasoning 模型，它的整个策略本身也就是它推演的执行了。

然后我觉得第四步它还有一个特别好的地方，就是它会做一定的反思、反馈，就是看一看最后的结果和目标之间的差异。这个方面其实它跟人做得非常像——人你要做一个面向客户、面向用户的业务的时候，你也要先做用户和市场的分析，接下来要针对用户来确定目标，然后第三步要有策略并且执行，第四步要有复盘。

我觉得它特别好地做了一个最佳实践。而且你可以想象，一个模型在做 reasoning 的过程、在做整个业务推理的过程中，其实是调动多个专家一起来做的。如果用到组织管理中，也是非常好的。

【李想】：一定是大家群策群力地去做分析，群策群力地去制定目标，群策群力地去制定策略，然后编辑起来做复盘。其实很多时候，包括复盘如果有问题，我们就进行改正。如果是业绩的差距，我就继续增加分析、调整策略；如果是能力的差距，我就去改善能力，继续走研究那条路径。对吧？因为很多时候，如果你能力不够，然后硬推着去做业务，其实也做不到。

我觉得第一个阶段，是我们从2011年开始，通过感知学习，配合后边的规则算法，通过规划、控制、执行，这些规则算法是分段式的。我觉得这个阶段，第一个阶段比较像昆虫动物的智能——它有既定的规则，还要依赖于高精地图，就比较像蚂蚁的行动和完成任务的方式。它能理解的世界也很有限，非常有限。

而且效率比较低，也是个很麻烦的事情。就是它就这样一个规模的脑子，包括它的整个模型规模，大概就只有几百万的参数，就那么小的一个脑子，你让它去完成复杂的事情，几乎是不可能的。所以你不停地限定，几乎把它做成了一个有规则交通的方式，这跟蚂蚁非常相似。

第二个阶段，就是我们从2023年开始搞研究、2024年推出的端到端。端到端比较像什么呢？端到端比较像哺乳动物的智能，比如像马、狮子这类动物，向人类学习怎么骑自行车，它学了人类的这些行为——人类怎么去做出各种各样的驾驶行为。但是它对物理世界并不理解，它只是看到了一个三维的图像，知道自身的速度，然后给出一个轨迹。所以它应付大部分场景是没问题的，但去面对它从来没有学到的、特别复杂的情况，其实就会遇到问题。

所以这时候我们也会配合视觉语言模型，就是VLM，放进来。但是我们能够用到的这些开源视觉语言模型，它在用于交通上的能力都非常有限，所以只能起到一些非常有限的辅助作用。我觉得这是第二个阶段，就是哺乳动物智能这样的一个运作方式。

我觉得到了VLA，就是完全人类的运作方式了。它会像人类一样，用3D视觉和2D的组合，去看整个真实的物理世界，也包含它能够看懂导航软件是怎么在运行的，而不是像VLM那样只能看懂一张图片。另外一方面，它有自己完整的脑系统，能够去——

【李想】：不但要看到物理世界，还能够理解这个物理世界。它有它的Language，有它的COT，然后有推理这样的一个能力。第三个，它能够像人类一样去真正地执行这些行动。我觉得这个其实就是VLA产生的原因。所以我们称之为——如果放在我们汽车的智能驾驶领域，我们把它称之为VLA的司机大模型。

如果你感兴趣的话，我跟大家讲一下VLA是怎么训练出来的，怎么去工作的。因为确实我听了你很多期节目，大家都在讲VLA，但我觉得大家讲得并不清楚，尤其到后边训练的时候，大家一笔带过，往往只是引用一些论文。我们是真正在干这事的，而且我们有足够多的数据，所以我们来讲一下。

【小俊】：您可以讲讲VLA这三个字母的关系是什么，以及怎么训练的？

【李想】：对，我来讲一下怎么训的，并把这个关系表达清楚。

第一个是训练的环节。训练环节的第一个部分，其实是要训出来一个VL的基座——就是Vision和Language结合的基座。我们目前在训的当前版本，是一个32B的，也就是320亿参数的，一个云端的基座模型。所以先训这个。

这里边跟过去那些语言模型的差异在于什么呢？第一，在于我要放入更多的Vision的数据，放Vision的token。这个Vision里边包含两个部分：一部分是3D的Vision，就是对物理世界的3D感知，这个要放进去；第二个是高清的2D的Vision。因为今天大家看到各种多模态的开源VLM里边，它的整个2D Vision的清晰度太低，所以看的距离不够。我们放进去的，感知距离基本上提升了3到5倍。我觉得这是非常重要的，这是Vision的token和数据的部分。

第二个是要放入Language部分的、跟交通跟驾驶相关的、足够多的这方面的数据。这是Language的部分。

第三个，还有一个很重要的——

【李想】：是大家可能容易忽略的——我们必须放入很多VL联合的预料，就是三维图像和对世界的理解、语义，要同时发生的。比如我举一个例子，我要把导航地图，以及对导航地图的理解，一起放进去。

【小俊】：这是原始的数据吗？

【李想】：不是原始的数据，这是我们自己的。所以我们才有……然后比如我要放入——其实一个人看到导航以后，人类做了一个什么判断，然后这个判断，我们的车辆是怎么执行的，要把这个预料放进去。

所以我觉得，整个VL的基座模型训练的时候，其实包含了三个部分：一个是Vision的数据，然后是Language + Vision的数据，和VL联合的数据。这样它就形成了VL的一个基座。

那同时，我要把这个基座拿来干什么呢？我要蒸馏下来，变成一个3.2B的端侧蒸馏模型。因为我要保证它运行速度足够快，以及它能够在——无论是两个Orin X还是所有骁龙上——可以流畅地运行。

所以我们蒸馏下来的是一个3.2B的、由8个专家组成的MoE模型。如果直接跑3.2B的一个完整模型的话，双Orin X和所有Orin的推理速率是达不到的，就是Token的整个输出率是达不到的。

所以这是第一个步骤，这是预训练的环节。

第二个部分，其实是要做后训练。

【小俊】：预训练是什么？

【李想】：预训练相当于学习人类的知识，就是先学习了人类怎么去理解物理世界、怎么去看物理世界、怎么理解物理世界，这样的一个知识。

【小俊】：蒸馏前是3.2B，对吧？蒸馏前是3.2B在云端——我一会还想讲，云端3.2B到后边还要继续去使用，OK？并不是只是把蒸馏基座放在那里了。

【李想】：对。那我觉得第二步骤是做后训练。后训练什么呢？后训练其实是我要把它变成VLA，我要把Action放进来。

这个后训练做的，仍然是一种模仿学习，特别像——特别像你去驾校学开车。你在驾校里学开车，就相当于……

【李想】其实我训练VLA，把它组合成一个VLA端到端的这样一个方式，我觉得这是……然后这个第二个部分，这个时候大概模型规模就会从3.2B扩大到接近4B，大概这么一个规模。

其实它一方面是个VLA，就是能够直接从视觉到理解、到最后的输出。但是比如我们的CoT就很短，我不会做超长的CoT，我的CoT链条一般两步到三步，我不会再做更多的，因为否则延时太长，没有办法满足交通或者机器人的安全要求。

另外，当Action做完以后，我还会做一个Diffusion的预测，就是下边会发生什么样的场景。这个主要根据性能，会做出来4到8秒这样的一个Diffusion的轨迹和环境的预测。所以这是第二个部分，比较像人去驾校学好开车这样的一个环节。

然后第三个部分是强化，要做强化训练——就像人到社会上开车了。强化我们分成两个部分。第一个部分先做RLHF，就是带有人类反馈的。我们有很多人类数据，就是当它这样操作的时候人类会接管，当那样操作的时候代表人类不会接管，包括人类的一些驾驶习惯，现在拿这块来做一个人类反馈的强化训练，大概是这个意思。包括我们的安全对齐，都是在这个强化的环节完成的。

就是你要遵守交通规则，你还要遵守比如中国大家的驾驶习惯，你的开车习惯能够融入社会。你首先要开得跟整个社会环境里大家一样好，你不能给别人带来麻烦，而不是像个新手在路上变成一个阻碍。所以这是强化的第一个部分。

然后第二个部分是纯粹的大规模RL，是什么呢？是我们拿世界模型生成的数据……

【李想】：训练这块的目的是什么？就是开得比人类更好。这块的话，我们中间不会给人类的反馈，只会给一个结果——就是从A点到B点开过去。但是中间会有三类训练要求：第一个是，我们可以通过G值来判断它的舒适性，给舒适性的反馈；第二个是要做碰撞的反馈，它碰撞了，这个强化就没有完成；第三个是交通规则的反馈，如果它违反交通规则就没有完成。所以是舒适性、交通规则和碰撞事故这三项，然后让它自己来做。

整个强化训练，当这三个步骤完成了以后，VLA能够跑在车端的模型其实就产生了。大概是这么一个过程——跟人非常像：先学习世界、交通和人类的这些知识，这是预训练的环节；后训练的环节相当于去刻意、认真地学开车；然后第三个环节相当于到社会上去开车，社会上的开车也是我们训练的一个过程，通过人类的RLHF来跟人类做对齐、跟社会环境来对齐，另外也利用我们自己在世界模型里生成的数据让它做强化训练，从而开得比人类更好，能够解决更复杂的问题，比人类平均水平开得要好得多。大概是这么一个过程。

【小俊】：我觉得这还没有完，有了VLA之后，人类是怎么跟VLA协作的？

【李想】：对，其实我还要搭建一个司机Agent。这个司机Agent是什么呢？就是人类以自然语言的方式——就是你跟一个司机怎么说话，你跟一个正常的驾驶员，假设你有男朋友，你男朋友在开车，你怎么跟他说，你就怎么跟这个司机Agent来说，就那么说就行了。

那这里边，如果是一些短指令、通用的短指令，会直接由VLA处理，不需要再经过云端；如果是比较复杂的长指令，其实是先要到云端那个32B的模型，因为它理解交通的一切，处理完以后再交给VLA来进行处理。

【小俊】：就大概是这么运行的一个过程。所以说白了，他最后的一个好处是，他能够像人一样去理解物理世界，能够像人类司机一样去开车、去处理复杂的问题，然后也能像人类司机一样跟其他人类进行沟通。我觉得这是最后我们交付到云端那里的产品。那这其中这些步骤里面，哪一个是最难的？

【李想】：没法预测。因为这些东西，前面没有任何人走过这条路。DeepSeek没有走过这条路，OpenAI也没有走过这条路，谷歌、微软也没有走过这条路。我们其实走的是个无人区。

【小俊】：那你们为什么就笃定压住了这条路呢？因为我最近做了一个技术播客，就是讲VLA，我就感觉这个技术路线还没有收敛，为什么你们觉得你们可以做？

【李想】：我觉得交通领域应该是VLA最早实现的。对吧？因为规则清晰——车的意识规则清楚，然后你说做强化学习是非常容易的。按照每个场景来讲，交通的世界里，一辆车会跑到哪里是有边界的。虽然它很复杂，但是是确定的。对吧？车不能开到水里，车也不能开到空中，车只能开到有路的地方。所以它的复杂性被确定性所约束。我觉得这是第一点。

第二呢，其实车的控制——车是个三自由度的系统。左右是一个自由度，前后是一个自由度，某种程度上还有一点轻微的旋转，是一个自由度。对吧？甚至自动驾驶撑死就控制两到三个自由度。如果这个都不能实现，机器人上来就是四十多个自由度，那个挑战就更大了。所以在这个阶段，我们进行模仿学习是特别容易的——车看到了什么，就是人看到的；人操作的，其实就是车操作的。所以第二个，它能做特别好的模仿学习。

第三个，它还能做特别好的强化学习。就是大家在使用的过程中，不满意的时候就接管了——

【李想】其实这就是跟人类没有对齐——就告诉你不应该这么做，以及告诉你该怎么做，这是非常清晰的指标。包括后面，我不做人类监督、靠生成式数据来做训练的时候，也非常清晰。因为什么是舒适，是可以表达的；什么是符合交通规则，是能够表达出来的；交通规则是个清晰的规则；是否发生碰撞，是可以表达的。然后如果它很舒适、又不违反交通规则、又没有发生碰撞，从A点到B点，它就会开得越来越好。我觉得这其实是VLA最好的、最重要的第一个实验场。

【小俊】我听你说完，有一个感受——刚开始大模型火的时候，大家都说创业要做AI是"造人"，你们这个其实就是在"造司机"？

【李想】对。而且我觉得有一个很重要的感觉：我们只有让它变成一个真正的司机，它才是一个生产力工具，而不只是一个辅助工具。今天R2、R2+其实还是辅助工具，辅助工具还需要人大量参与。但我觉得如果想变成生产工具——我个人认为并不会出现通用的Agent，而是每个专业领域做专业的Agent。就像我刚才讲的，想开好车，它所有的Vision语料、Language语料和Action其实都是不一样的；你想做好一个医生、做好一个律师，包括背后整个思维链、背后所有的数据，其实都是完全不同的，所以我是这样判断的。另外一方面，如果你想变成一个生产工具，这时候就会和专业的人去比较——你是否比一个专业的司机开得更好？你是否比一个专业的医生表现得更好？你是否比一个专业的律师表现得更好？你是否比一个专业的程序员表现得更好？因为你会影响到他们整个的生产。

【李想】然后它的整个业绩、工作结果，甚至财产和生命安全，我觉得这是我们看到的终点。所以这是为什么我们必须很耐心、很深入地去解决。哪怕只是一个司机这样的问题，也并不可能通过一个泛化的大模型，或者一个通用模型，这些东西都能实现。我觉得这是不现实的。

【小俊】为什么通用 Agent 不如专业 Agent？在你看来，我们需要通用 Agent 吗？

【李想】我觉得通用 Agent 换个描述方式，比较好的描述方式应该是 Agent OS。我觉得五年之内不会有通用 Agent，但会有一个 Agent OS，方便各个专业的人在这个 Agent OS 上开发出自己需要的 Agent。我觉得这个反而是真正需要的。

【李想】对啊。因为你要做一个 Agent，或者说你要做一个生产工具，就是要能够替代和解放真实的工作——每天那些高频的工作，比如开车，比如编程，比如很多事情——而不只是把信息索引一下、在脑子里转一下就结束了。我觉得这是根本性的不同。而且这时候，就像我刚才讲的，它会影响到你的收入，它会调用你的资产、你的工具，或者你的财产——你的电脑、你的车，对吧。这时候我觉得整个对齐的要求也不一样了。比如你今天拿 DeepSeek 或者 OpenAI 去查找一个信息，它就用 RAG 的方式去做索引；但你拿 DeepResearch 或者 Manus 这样的，还会去找信息的源头，而不是简单地通过 RAG 就可以。因为专业人员一定是找信息的源头，找到那个最真实的源头信息，才能往下做分析。

【小俊】你们是不是也要做 Agent OS？

【李想】我觉得我们做的 Agent OS，想来想去只有司机这一个，其他的我们不专业。你说我们做教育的……

【李想】我觉得那应该是原福岛——好未来他们去做的，对吧。最后AgentOS会搭载在我们车上，这是有可能的。但是我觉得我们比较重要的是，我们先做好公司内部的AgentOS，因为公司内部需要很多的AgentOS——有客服的AgentOS，对吧，刚才讲的有编程的AgentOS，对吧。
