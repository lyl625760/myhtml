【吴明辉】：我当时正好在北大读博士，是19年回北大又开始读这个工程博士。我当时读的方向其实就是我们叫conversational intelligence，其实是自然语言处理里面的一个更小的分类，研究人和人对话过程中产生的智能，然后AI去理解这个对话，甚至可以生成这个对话，其实跟后来ChatGPT是有关系的。

【小俊】：就是你拿了这笔钱就开始做自己的产品了？

【吴明辉】：因为在这之前，我其实一直都是给别人做软件开发、做外包的。那为什么后来我们的商业计划书会去写 Reconciliation System？一个很重要的原因是，我在给新浪、和讯等做软件开发的那个年代，整个互联网经历的正好是 Web 2.0。咱现在讲 Web 3，Web 2 和 Web 1 最大的区别是：Web 1 是单向广播式的；Web 2 才出现了基于每个人的 ID、来做这种双向交互类的应用，比如说博客啊、email 啊。所以 Web 2 比 Web 1 就多了一个 user 的 ID，使得每个人可以看不同的内容——Web 1 所有人看的新闻都是一样的，对吧。

所以其实当时 Web 2 比 Web 1 也增加了巨大的技术复杂性。原因是：一套所有人看内容都一样的系统，如果用户量很大，它是可以通过一层一层的缓存来承压的，因为每个人看的都一样，中间有几个中间服务器把流量分开就行了。但是当每个人看的是……

【吴明辉】读属于你自己的内容的时候，这个时候其实我就不能随便做缓存了。你只能看你的嘛，所以我把你随便分到一个服务器，那个服务器上没有你的内容，那实际上你就用不了了。

所以那个切入点，其实我们的团队技术真的是很厉害的。所以我带着这个团队就给新浪啊、搜狐啊，就提供这方面的 infrastructure。实际上这个 infrastructure 最早是在哪出现的？就是 BBS 嘛。BBS 每个人登进去，肯定要看自己的内容，你发自己的内容嘛。所以其实我们有这样的一个能力。

后来我们融资的时候呢，很多投资人就问说，你们公司到底有啥技术啊？做前教育的话，我就开始讲嘛。我说我们能给不同的人推荐不同的教育教材，是吧，就所谓的因材施教系统。当然后来别人也没买单，但是就是把这样的一个技术概念，在那个时间点稍微归纳了一下。

就这样，我就在学校里面——所以我的硕士毕业论文写的应该叫 recommendation system based on language model。虽然我研一研二做的是图像，那个时候其实视频也做不了，因为显卡都不够大，但是我研三其实已经在做 NLP 了，做 recommendation system。

【小俊】你为什么是 based on language model？这个非常有意思啊。

【吴明辉】这个其实没有"large"，是没有大语言模型，就是语言模型。语言模型其实在北大是一个非常非常早就有的东西。北大机器系跟北大中文系中间有一个联合研究所，叫北大自然语言处理研究所。这个研究所很早就做了一个工作，叫CCD，叫中文概念词典。

概念词典是什么呢？它其实就是建立了一个数学上叫二部图的结构，就是一边是所有的单词（word），另一边叫word sense，其实就是每个词的词义。比如说"一词多义"，就是一个单词对应好几个词义；那"同义词"呢，其实就是好几个单词对应同一个意思。那其实这个就是所谓的语言模型——语言模型其实就是把所有的词构建到一个空间里面，一个向量空间里面，去看它们之间的关系。

所以最早最早的语言模型，其实就是由人来标注词和词、词和词义之间的关系，这就是以前的做法。但是后来有一些重要的工作，像word2vec之类的，就把这个东西给颠覆了。当然这都是我毕业以后发生的事情。

我读书期间，北大就有这样一个概念词典。那上一代的推荐系统（recommendation system），其实都是基于人的浏览行为的相似度来做推荐的。经常说的就是，我看了一个网页，或者我看了一个商品——最早讲推荐系统就是这么来的——所以它其实是说"看了这个商品的人还看过什么"，然后就可以推荐给类似的人。

但是这里面也有一个问题：如果一个新来的用户只看了一个网页，信息量很少，也不一定能给你推荐得非常准确。还有一个很大的问题，就是商品和商品之间的相似度其实是不知道的。

所以我们当时做的，为什么要基于语言模型——就是我会去看两个网页之间的相似度。不是说同一个网页，如果两个网页之间也有相似度，那也就意味着：如果一个人看了A网页，另外一个人看了B网页，但A网页和B网页本身是有相似度的，这样的话我也可以把推荐的范围拉过来。所以这个就是我的硕士论文。

而且我的第一个用户，其实是因为李峰的原因，所以是牛博网——我不知道你听没听说过，就是博客类的网站嘛。因为当年他曾经当过这个公司的股东，所以我的这个产品当时就放在牛博网上做推荐。

【小俊】你为什么没有做一个像今日头条那样的产品？当然它比今日头条成立还早好几年。

【吴明辉】好问题。

【吴明辉】我的第一版商业计划书写的就是这个。就是从那个教育业务把它干掉了以后，而且李峰离开了以后，我的融资商业计划书写的就是推荐系统。但是太早了。其实后来做很多事我就发现，我有时候做得太早了，真的会有很大问题。

【吴明辉】07年出来讲这个故事的时候，iPhone 1还没发布呢。iPhone 1好像是07年12月份发布的。所以在那个时间点上，如果只有PC互联网的话，Recommendation System只能由一个公司拥有自己的海量用户、且拥有注册用户才行。因为PC互联网，用户是没有一个唯一标识ID的，它不像移动互联网。

【吴明辉】我觉得张一鸣之所以成功，一方面技术肯定是越来越先进，但更重要的一点是，它诞生在移动互联网上。移动互联网和推荐系统天然契合，因为你做一个新的app，或者做一个新的网站，没有人愿意过来注册，对不对？除非像QQ这种。在没有人愿意注册的情况下，如果你没有一个人的唯一ID，你就没办法持续积累这个人的historical行为数据。你如果积累不了历史行为数据，那你就没办法做下一步的推荐。

【吴明辉】我们在2007年就讲这个故事，但实际上基础设施是不ready的，所以也是那个原因就做不起来。PC互联网时代有一个东西叫Cookie，以前是在Cookie里面设了一个ID，然后去持续记录一个人的连续行为。但那个Cookie特别不幸，在美国一两个月就会被像360这样的软件清一次，所以这个ID不稳定，就做不了这个事情了。所以推荐系统是到了移动互联网之后才真正跑起来的。

【吴明辉】但是到了移动互联网的时候，我们原有的业务已经做得非常好了，就有点忘记初心了。本来我们做广告数据分析，是因为想积累数据、再去做Recommendation System，对吧？但是结果等到那个业务做得很好、开始赚钱了，移动互联网又来了。我觉得那时候我可能商业敏感度不太够，我第一个手机其实是到iPhone 4才买。所以在这件事情上，我是错过了一个时代。

【吴明辉】如果我在2011年、或者2012年开始好好去整Recommendation System，其实我们的优势是比今日头条要好的，因为我有比它多的数据。但是再过两三年就没用了，人家就太厉害了。

【小俊】你当时想把推荐系统应用到什么样的场景？

【吴明辉】最一开始其实是契机推荐。但是就像我上面说的，第一版是先做广告数据分析，第二阶段再去做广告的Recommendation，第三阶段才是推荐任何信息。

【小俊】还是从你们当时已有的业务去触发的？

【吴明辉】对对对，因为我当时已经在广告数据分析上开始做起来了。

【小俊】李峰为什么07年就走了？是07年秋天吗？

【吴明辉】06年开始的。

【小俊】是06年？

【吴明辉】对，可能都不到一年吧。

【吴明辉】我觉得最核心的是，他觉得这个事好像跟教育没什么关系，他还是想做教育。我觉得他可能不是说一定要想做教育，他更重要的是一定要把自己最大的价值发挥出来。如果不做教育的话，他好像对这个公司最大的价值就不大了。他作为一个CEO——你想，他也不是一个Sales那种路子的人，所以他后来就去做投资了。

【小俊】他是离开以后决定去做投资？

【吴明辉】对。就是他当时觉得，他自己本人在这个公司作为CEO，contribution不一定很大，他说还不如直接去到资本市场。其实他走了以后，他还是公司的股东，当然他也让出来了一些股份，给到了我后来的合伙人。所以他让出来一些以后，我才变成了第一大股东——我的股份并没有增厚，是他拿了一些股份出来，就像是公司的股权变得更散了。

【小俊】所以核心是你们三个人，对吧？

【吴明辉】对，是。

【小俊】这很有意思，我很好奇。就是因为你也是学霸，他也是学霸，学霸组合起来一起创业，你觉得合适吗？

【吴明辉】我觉得如果是同一个领域的学霸的话，可能是有风险的；如果是不同领域的话，是没问题的。其实你看乔布斯去搭一个团队，他就希望有一个硬件的天才，有一个软件的天才，也有可能有一个供应链的天才。天才和天才之间是心心相惜的——就是因为你那个事特别牛，我也不懂，但你说一句话我信；我这个事你可能也不懂，那我说一句话你也信，那大家就可以一块很好地合作了。但是如果大家比如说都是做软件很厉害，那还是很容易文人相轻的。所以我觉得，互补的团队还是蛮重要的。

【小俊】你们是match的吗？你们是互补的吗？

【吴明辉】比较互补的。因为其实我的东西他肯定是不懂的，只不过他最后觉得，他自己特别懂的方向，可能对这个公司不一定有价值了。

【小俊】我们刚才是从15年讲到这儿的，因为我们倒回去讲一讲吧。你之后一路都是保送的？

【吴明辉】对，我基本上是一路保送的。我是一个相对比较偏科的人，数学、物理、化学都很好，语文、英语一般吧。但后来我觉得语文、英语在工作过程中确实越来越重要了。读书的时候，基本上数理化几乎都是满分，其他的我也无所谓。

第一次参加数学竞赛培训，是小学五年级，当时老师给了我一个很大的震撼。

【吴明辉】老师就说，同学们你们之前——因为我们被选到了学校的数学代表队——老师就说，你们之前这个学习时间都花得比较长，其实没必要。他说："我告诉你们，其实这一学习课你们一个礼拜就能学完。"我当时听了也觉得挺夸张的，但是一个礼拜就学完了。学完以后我们也都能考得很好，所以我后来才发现，原来学校的课程其实不应该那么慢地去学的。

所以后来我初中就把高中的数学、化学全学完了。所以我高中是不上课的，我高中每天——老师都是把学校实验室的钥匙给我，说"你爱去哪个地方自己去"。当然我更多是骑自行车去我们市图书馆，每天就去看各种各样的数学书。我是真的喜欢数学。

那时候计算机也还不错，因为数学、物理、化学、计算机全都是每一轮参加竞赛都拿很好的奖。

【小俊】那你成长路上其实没有什么坎儿，对吧？就是都很顺的？

【吴明辉】读书期间肯定是没有的，就是从小都是每天老师喜欢的那种，老师不会做的题然后我来给大家讲那种。上大学以后也是这样子的，不过上大学稍微有一些变化。

本来我每一科都参加竞赛，第一优先选择本来是想去清华计算机的。但是特别机缘巧合——数学竞赛先考嘛，考完以后我当时是山东省第二名，全国可能是二十一二名的样子吧，反正就是差一点点进国家集训队。不像现在——如果现在考的话就能进集训队，因为现在集训队一下招60个人，当时我们只招20个人。

我当时数学竞赛考完以后，全国数学竞赛前60名是可以随便挑清华或者北大的。我本来确实是直接想去清华计算机的，但是北大的招生办老师特别狡猾——他就跟我们山东省队的前三名的学生说，因为我们省队一共五个人，前三名的学生可以在清华和北大之间选。然后他说："如果你们前三名都来北大，那我就给剩下两个同学发offer。"因为剩下两个同学如果我们不帮他们的话，他们两个就去不了这两所学校了。所以那不能害了自己的队友，所以我就去北大了。

去了北大就不可能去计算机了，北大肯定第一系就是数学系嘛，所以我就去了数学系。但是去了北大实际上压力是不小的，因为北大数学系号称是"北大四大风人园之首"嘛。所以去到那里，虽然我从小到大都是别人家眼里的那个孩子，但是到了北大你就是一个很平庸的——我觉得可能是我对自己要求高一些吧。就是可能比如说两百个人，我还是能考到二十名、三十名之类的，但是就不是第一名了，是吧。

因为我们那一届很夸张，有几个大数学家，号称是北大数学的黄金一代。

【吴明辉】所以我跟他们一比，我就不是那种天才型选手了。所以我当时就选择了一个差异化路线——我说那我数学是考不过你们了，那我就开始创业赚钱，然后开始写代码。因为写代码我在北大也都是最厉害的，所以因为有了一些压力，又选择了一次差异化定位。

【小俊】你在大学开始发现自己不一定能考第一名、第二名的时候，对你来说是一个挫折吗？还是说你很快就调整了，即使挫折也很快调整了？

【吴明辉】搞数学竞赛的人，不会特别struggle一件事马上解决不了。数学竞赛是一个特别好的训练，或者说所有的奥林匹克类的运动——就不光是数学，可能羽毛球、篮球也都一样——只要是你要做奥林匹克类的项目，本质上你要把一件事卷到全世界最牛，是吧？就跟今天做AI的训练一样，你要卷到最牛。其实一旦卷到最牛的话，它就一定是挑战极强，而且天天失败的。

就像打大比赛。我前两天正好跟南顿的总教练王指导——王浩聊天，他说运动员其实都属于很难抑郁的，就是因为你每天都在打比赛，赢输赢输很多，它不会输一局就怎么着了。做数学竞赛也是一样，就有太多的难题了。因为它不像日常的题——日常题对我们这些人来讲，没有任何一道题是不会做的，你不得满分都是不对的。但是奥林匹克的题，不会做很正常。

我记得，因为我儿子、我女儿现在都在打奥赛，然后我特别喜欢英国数学奥林匹克的卷子。它的第一页上面有一个说明，就说这个卷子很难，说你只要有一道题能完整地做出来就已经很好了。所以其实相当于你从小接受了这样一类训练——难题搞不定也很正常，就不要suffer，你已经很优秀了。

所以从这个角度来讲，你搞不过别人肯定是不好的，但是你从小的训练，因为你天天做难题，你也觉得很正常。

【小俊】他们说你乐观，是这个原因，是吧？

【吴明辉】对，我是超级乐观的。基本上没有什么事能让我持续不舒服。有的时候一些事会让我有那么一两秒钟不舒服，但是真的那一两秒钟之后我就过去了，这页就翻过去了，很快。

【小俊】这个跟竞赛是有关系的？

【吴明辉】这个我没有意识到，但我觉得跟竞赛是有巨大的关系的。

【小俊】当时有没有意识到，你毕业后会创业吗？

【吴明辉】我觉得人生很多选择随机性很大。你看我创业也不是一上来就plan好的，正常情况下我可能也会去找一个大公司工作，拿个高薪之类的。

创业其实真的就是——数学竞赛背景的学生，一进去很容易就被周围的培训机构给盯上了。当时在北大盯得最多的就是人大附中的，以前我们叫"华罗庚数学学校"，后来现在不知道叫什么名字了。总之它基本上每年都会去北大找奥赛保送生来做奥数教练。所以其实一入学大家就有这样一份工作可以赚钱。然后后来我在那个地方讲得特别好，我讲课还真的讲得不错。

【小俊】：对，你口才很好。

【吴明辉】：是的，我觉得我这个天赋就是"说"。我在中学就被练出来了，因为老师经常让我讲。我们学校的数学竞赛培训，大概就选十个人，学校里数学最好的十个人。我们校长是学数学的，他请了他一个大学同学过来给我们当奥赛教练。但那个大学老师数学特别牛，就是不会讲，所以基本上每次他把题做完了，由我来讲。所以我真的是从中学就开始锻炼——给别人讲复杂的题。讲难题，简单题我也没那么感兴趣，就是复杂题怎么把它抽丝剥茧讲得特别细，让所有人都明白，甚至小学生都明白。这个是我的天赋，或者说是从小练出来的吧。

但我最后真正开始做软件公司，也是非常机缘巧合。因为我家历史上是没有商人的，没有经营企业的人。就是我的一个中学同学，他是跟我一起搞计算机竞赛的——因为我计算机竞赛也做了很久，我是从初中二三年级开始参加计算机竞赛的，上大学之前连续参加过五届信息学奥赛。他当时是我的队友，是我们学校另外一个同学。他拉我入伙，就是一起接活赚钱。他其实编程没有我厉害，但挺能折腾的，他就出去接这种外包项目赚钱。但是发现他的合作伙伴有很多Bug搞不定，然后他就过来请教我，说这些Bug能不能搞好。我就到图书馆里找了一些书研究了一下，然后说："来，你把代码交给我吧。"我一天就给他改完了。改完以后他说："要不咱们一起来做吧。"

所以其实是非常非常随机地开始做企业的，不是一上来就plan好的。

【小俊】：当时第一个创业，后来为什么做的是一秒针系统？

【吴明辉】其实就是因为做Recommendation System，没有数据。经历了一个很大的兜兜转转，最后发现原来帮客户做广告数据分析是可以获得数据的。

我最开始的想法，本来是想做一个流量数据分析系统加推荐系统，给到媒体。后来发现媒体都不喜欢我们这个系统。为什么呢？我一开始也不知道。后来有一个跟我关系挺好的朋友用了我的系统，我才知道——是因为大部分媒体在那个时间点上，对外卖广告没有一个特别好的方法，就用最简单的方式：吹自己的流量有多大。实际上真实的流量跟他们对外讲的差很大，所以他们不希望一个第三方公司知道真实的流量。

【吴明辉】所以我在那个时间点上就发现了这样一个商机。我就去找那些广告主，因为他们每天在买广告，其实一直在被骗。如果我们能帮他们知道真实的流量，他们会很高兴。

第二个阶段，才是师父觉得说还是要好好地给徒弟讲一讲，所以他就有一个叫Externalization，就是外显化。这个时候，通常比如说公司内部的一些专家的分享、知识的交流，大家会写个PPT出来讲一讲，这个事情到底怎么做——第一步、第二步、第三步。这个过程，其实我们讲外显化。

【吴明辉】就是个E的过程。但是光有E不行。为什么？每个企业的知识还涉及到同一个事情，可能有好几个不同的专家，那专家和专家的想法可能还不一样。所以就来了第三个环节——C，就是Combination，就是连接。

这个连接，其实在一个企业内部，比如干同一件事，到底有不同的办法，到底哪个更好、哪个不好，哪一种情况应该是A、哪一种情况是B，所以需要把几个不同的专家的外显化的知识再去做一次连接。其实我们学术圈里面，大家把各种各样的文章之间互相比较、引用，其实也都是在做这个事，是吧？就分各种各样的情况讨论说，哪一种情况应该用这种模型，哪一种情况应该用那种模型。

好，最后一个环节，其实就是I的环节，就是Internalization，就是内化一个知识。比如说当我们的知识被外显化了，经过最后Combination之后变成标准化了，一件事情到底应该怎么干，已经有比较清晰的结论、有这个guideline的时候，就是每个人再去学习它，然后再把它变成自己脑海里面可以执行的任务知识。

所以这个就是一个SECI的模型。那这个过程其实是一个循环——人脑的隐性知识传递到另外一个隐性知识，人脑的隐性知识又变成外显了、变成显性知识，显性知识又和显性知识进行连接，然后最后显性知识又内化到了脑子里面，变成了一个隐性知识，整个就是这样的一个过程。

这里面其实真正最难的部分，就是这个隐性知识到底怎么变成Memory——变成我们今天AI可以用的Memory。因为它在人的脑子里面，人如果不告诉AI就是不知道的。而且它每天都在叠加，有的时候跟外部互动产生叠加，有的时候就是它脑子里面在静思的过程中也可以叠加，叠加在脑子里面的这个隐性知识。

所以我觉得真正有效的方法，一定是采用类似DeepMind也好、Manus也好这种人机互动的——而且最好是持续互动的这种方式。最理想的是一个知识工作者，他所有的工作都是在这个平台上做的。他在平台互动的过程中，在执行最后完成任务的过程中，他第一步的那个Prompt，有可能都没有把所有的信息都给到。但没关系，你出的结果不对，我接着再追加问题，一直改。其实这个人机协作的过程中，AI就把你脑子里面完成任务所需要的隐性知识也都挖出来了——只要最后的结果是对的，那肯定你已经把所有的context都给到了，是吧？

就像——咱们现代智能体有一篇非常著名的，其实不是paper，是个博客，就是那个Lilian写的。她最近不是跟那个谁，跟Mirror一起做Thinking Machine嘛，对，那家公司的联合创始人，她也是我北大的师妹。这个Lilian其实原来写过一篇非常有名的博客，叫《LLM Powered Autonomous Agent》，就是基于大语言模型驱动的自主智能体。

但是你想，它强调的是——今天因为AI有了planning能力，有了大语言模型就可以有planning能力了，有了planning能力它就可以自动分解任务、自动执行、自动闭环。但是这里面如果是完全autonomous，它其实就没有能力去挖人的context，是吧。

所以其实如果我新写一篇paper的话，我肯定会去写的是一个基于大语言模型的Human-in-the-Loop的agent，甚至是一个moldable agent。那它这里面的协作AI，到底怎么问人类问题？怎么能够把人真正想说的问题问出来？所以这个是很关键的。

所以我自己的产品设计里面是有两个模式，一个是自主模式，就是autonomous agent；一个是Human-in-the-Loop。

【吴明辉】你如果启动这个模式，你会发现它就很烦。这个跟 Manus 完全不一样——Manus 你过去给它一个任务，它马上就啪地干了，一波流就出来了。但是我一旦用这个模式的时候，它会不停地问你问题，直到它觉得该问的都问够了，它才干活。而且干的过程中，你还发现它干得不对，你可以接着再去改。

以及，因为在 data 这个赛道里面，我前面讲了，即便是我们今天已经做了大量的优化，仍然有少量的幻觉。所以我们其实是把我们整个产品的工作流程里，每一个跟数据相关的处理细节，全都更加精细化地透明出来了。

就 Manus 其实它也做了大量透明，是吧，你能看到它执行任务的过程。但是涉及到 data 的，它非常不透明。就是你如果让它做一个类似于 deep research 的任务，你其实也不知道那些结果最后是怎么算出来的，A 到 B、B 到 C，它没有太多的……如果你仔细去看它执行任务的中间过程，有很多任务其实是在写代码的时候，现编的数——你看一下，它执行调用的时候，它写了一个代码，那个代码里面已经把那个 chart 的数给编出来了，然后最后画出了一个 chart。就是这样会产生很多幻觉。

所以我们也发现，这种情况可能是不可避免的。那你怎么办？那涉及到所有的数据，全部都以各种 CSV 或者各种各样的形式全都透明出来，然后我们的客户在上面，可以在任何一个环节里面去做修改、可以做 check，甚至可以做溯源。所以这个其实就会使得我们的人能够更多地敢于去信任这个工作，然后人机协同的效率也就会更高。

总之就是，如何把人脑子里面的 context，通过这种多轮的互动过程抽出来，这个其实是我们产品的一个重要理念。
